# StreamWeave Chrome 插件产品需求文档

版本：v1.0  
最后更新：2026-06-14  
适用范围：Chrome / Chromium 浏览器插件 V1，配合 StreamWeave 桌面 App 使用

## 1. 背景

StreamWeave 当前通过桌面 App 输入 URL 并加入下载队列。用户在浏览器中浏览视频网页时，需要手动复制当前页面 URL，再切换到 App 粘贴，操作链路较长。

Chrome 插件的目标是把“浏览器中正在看的页面”快速导入 StreamWeave：用户可以一键导入当前标签页，也可以从已打开标签页中勾选多个 URL 批量导入。

V1 插件只采集用户明确选择的标签页 URL，不读取 Cookie、不抓取网页内容、不注入脚本、不绕过登录或 DRM。

## 2. 目标用户

- 主要用户：已经使用 StreamWeave 桌面 App 的 Chrome / Chromium 浏览器用户。
- 使用场景：用户在浏览器中打开多个视频网页，希望快速加入 StreamWeave 下载队列。
- 典型输入：当前活动标签页 URL，或用户勾选的一组已打开标签页 URL。
- 典型输出：StreamWeave 桌面 App 中新增一个或多个下载任务。

## 3. V1 产品目标

- 支持一键导入当前标签页到 StreamWeave。
- 支持列出已打开标签页，并批量勾选导入。
- 使用 macOS 自定义协议唤起 StreamWeave，不依赖本地 HTTP 服务或 Native Messaging。
- 插件权限保持最小化，降低用户安装顾虑。
- 导入失败时给出明确提示，例如 App 未安装、URL 不支持或选择数量过多。

## 4. V1 非目标

- 不读取 Chrome Cookie、浏览器历史、表单内容或网页正文。
- 不注入页面脚本，不解析网页中的 m3u8。
- 不直接下载视频，下载流程仍由 StreamWeave 桌面 App 负责。
- 不做 Native Messaging、本地 HTTP 服务或浏览器与 App 的双向长连接。
- 不支持 Safari、Firefox 插件。
- 不支持上百个 URL 的大规模导入。

## 5. 核心用户流程

### 5.1 导入当前标签页

1. 用户在 Chrome 中打开视频网页。
2. 用户点击 StreamWeave 插件图标。
3. 用户点击“导入当前标签页”。
4. 插件读取当前活动标签页 URL。
5. 插件打开自定义协议链接：

```text
streamweave://add?url=<encoded-url>
```

6. macOS 唤起 StreamWeave。
7. StreamWeave 校验 URL 并加入下载队列。
8. 插件显示“已发送到 StreamWeave”。

### 5.2 批量导入已打开标签页

1. 用户点击插件中的“批量导入标签页”。
2. 插件显示当前窗口或全部窗口中的普通网页标签页。
3. 用户勾选需要导入的标签页。
4. 用户点击“导入所选标签页”。
5. 插件将 URL 数组编码为 base64url JSON，并打开自定义协议链接：

```text
streamweave://import?urls=<base64url-json>
```

6. StreamWeave 解析 URL 数组，按顺序加入现有任务队列。
7. 插件显示导入请求已发送。

## 6. 功能需求

### 6.1 插件入口

- 插件使用 Manifest V3。
- 浏览器工具栏显示 StreamWeave 图标。
- 点击图标打开 Popup。
- Popup 包含两个主操作：
  - “导入当前标签页”
  - “批量导入标签页”

### 6.2 当前标签页导入

- 插件读取当前活动标签页。
- 只接受 `http://` 和 `https://` URL。
- 不接受以下页面：
  - `chrome://`
  - `edge://`
  - `about:`
  - `file://`
  - 空 URL 或未完成加载的标签页
- URL 必须使用 `encodeURIComponent` 编码后写入 `url` 参数。
- 插件通过打开 `streamweave://add?...` 完成导入。

### 6.3 批量标签页选择

- 插件可以列出当前窗口标签页。
- 后续可扩展为“当前窗口 / 全部窗口”切换。
- 列表项显示：
  - 页面标题
  - 域名
  - 完整 URL
  - 勾选框
- 默认勾选当前活动标签页。
- 已选数量为 0 时禁用导入按钮。
- 非 `http://` / `https://` URL 不可勾选，并显示“不支持”。
- V1 建议限制一次最多导入 20 个 URL。

### 6.4 批量导入协议

- 批量导入使用：

```text
streamweave://import?urls=<base64url-json>
```

- `urls` 解码后必须是 JSON 字符串数组。
- 数组中的每一项必须是 `http://` 或 `https://` URL。
- 插件需要在打开协议前估算最终协议 URL 长度。
- 如果协议 URL 过长，插件提示用户减少选择数量。

### 6.5 StreamWeave App 配套能力

- App 注册 `streamweave://` 自定义协议。
- App 支持两个动作：
  - `add`：读取单个 `url` 参数。
  - `import`：读取 `urls` 参数并解析 URL 数组。
- App 收到 URL 后复用现有“加入队列”逻辑。
- App 保留现有重复 URL 处理策略。
- App 已运行时，协议导入应进入当前实例，不重复打开多个主窗口。
- App 未运行时，协议应唤起 App 并在初始化完成后导入任务。

### 6.6 插件权限

- 当前标签页导入使用：
  - `activeTab`
- 批量标签页列表使用：
  - `tabs`
- 打开自定义协议使用浏览器能力，不需要本地服务权限。
- V1 不申请：
  - `cookies`
  - `history`
  - `webRequest`
  - `scripting`
  - `<all_urls>` 内容脚本权限

## 7. 交互与界面需求

### 7.1 Popup 首页

- 顶部显示 StreamWeave 名称和简短状态。
- 主按钮：
  - “导入当前标签页”
  - “批量导入标签页”
- 底部显示帮助文案：
  - “需要先安装 StreamWeave 桌面 App”
  - “插件只会导入你选择的标签页 URL”

### 7.2 批量导入页

- 显示标签页列表。
- 支持全选 / 取消全选。
- 支持按当前窗口展示。
- URL 过长时列表项允许省略显示，但鼠标悬停显示完整 URL。
- 底部固定操作区：
  - 已选择数量
  - “导入所选”
  - “返回”

### 7.3 反馈状态

- 成功打开协议链接后提示：
  - “已发送到 StreamWeave”
- URL 不支持时提示：
  - “当前标签页不是可导入的网页 URL”
- 未选择标签页时提示：
  - “请选择至少一个标签页”
- 选择过多或协议 URL 过长时提示：
  - “选择的标签页过多，请减少数量后重试”

## 8. 异常处理需求

- App 未安装或协议未注册：
  - 插件提示用户安装或打开 StreamWeave。
- App 已安装但未响应：
  - 插件提示用户手动打开 StreamWeave 后重试。
- URL 编码失败：
  - 插件提示导入失败，并保留当前选择。
- App 端解析失败：
  - App 任务日志或系统通知显示失败原因。
- 批量导入部分 URL 无效：
  - App 跳过无效项，并提示成功导入数量和失败数量。

## 9. 安全与隐私边界

- 插件只读取用户当前操作范围内的标签页 URL。
- 插件不读取 Cookie、不读取网页 DOM、不读取表单字段。
- 插件不上传 URL 到任何服务器。
- 插件不做 m3u8 解析，解析仍由 StreamWeave App 完成。
- 对需要登录的网站，登录态仍由 StreamWeave 自己的 Playwright Profile 处理。
- 本功能不用于绕过 DRM、登录授权或访问控制。

## 10. Roadmap

### V2：更稳定的批量导入

- 如果用户需要一次导入大量标签页，改用 Native Messaging。
- 支持插件和 App 双向确认导入结果。
- 支持导入后在插件中显示任务创建结果。

### V3：浏览器侧辅助识别

- 在不读取 Cookie、不绕过授权的前提下，考虑对当前页面 URL 做简单分类。
- 可以标记疑似视频页面，但不在插件中解析 m3u8。

## 11. 验收标准

- 当前标签页为普通 `https://` 视频网页时，点击插件后 StreamWeave 新增任务。
- 当前标签页为 `chrome://` 页面时，插件拒绝导入并显示明确提示。
- 批量选择 3 个普通网页标签页后，StreamWeave 按顺序新增 3 个任务。
- URL 包含中文、query、hash 和特殊字符时，App 能正确收到完整 URL。
- App 未启动时，自定义协议可以唤起 App 并导入任务。
- App 已启动时，导入任务进入当前队列，不重复打开多个窗口。
- 插件权限列表中不包含 Cookie、网页内容脚本或网络请求拦截权限。
