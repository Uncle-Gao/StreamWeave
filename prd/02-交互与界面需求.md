# Video Downloader 交互与界面需求

版本：v1.0  
最后更新：2026-06-13  
适用范围：Tauri 桌面版 V1，本机自用，优先 macOS

## 1. 界面目标

V1 界面服务两个入口：输入 m3u8 URL 直接下载，或输入普通视频网页 URL 后自动提取 m3u8 并下载。界面由 Tauri Web UI 承载，应清晰展示任务状态，不做复杂导航，不做营销页。

## 2. 主窗口布局

主窗口采用单页布局，建议宽度 760-900 px，高度 560-700 px。

### 2.1 输入区

- URL 输入框
  - 单行输入
  - 支持粘贴长 m3u8 URL 或普通视频网页 URL
  - 输入为空时禁用开始按钮
- 输出目录选择
  - 默认 `~/Downloads`
  - 提供“选择...”按钮
- 输出文件名
  - 默认从 m3u8 文件名或 URL hash 生成
  - 自动追加 `.mp4`

### 2.2 操作区

- 开始下载按钮
- 取消按钮
  - 仅任务运行中可用
- 打开输出文件按钮
  - 仅任务完成后可用
- 在文件管理器中显示按钮
  - macOS 下在 Finder 中显示
  - 任务完成或失败但已有输出目录时可用

### 2.3 状态区

- 当前阶段文本
- 总进度条
- 分片进度：
  - 总分片数
  - 已完成分片数
- 当前速度
- 输出文件路径

### 2.4 日志区

- 多行只读日志面板
- 自动滚动到底部
- 支持复制
- 失败时显示错误摘要并保留完整日志

## 3. 状态机

V1 使用以下任务状态：

- `Idle`
  - 未开始任务
  - 可编辑 URL、输出目录、文件名
- `Resolving`
  - 检查工具依赖
  - 下载并解析 m3u8
- `Downloading`
  - 下载 ts 分片
  - 展示分片进度
- `Merging`
  - 调用 ffmpeg 合并 mp4
  - 禁用输入
- `Verifying`
  - 调用 ffprobe 验证输出
- `Completed`
  - 显示结果摘要
  - 允许打开输出文件
- `Failed`
  - 显示错误原因
  - 允许重试
- `Cancelled`
  - 用户取消
  - 保留日志和临时目录提示

## 4. 文案规范

### 4.1 阶段文案

- 准备中
- 正在获取 m3u8
- 正在解析网页
- 正在静态扫描
- 正在捕获网络请求
- 正在解析分片
- 正在下载分片
- 正在合并 mp4
- 正在验证输出文件
- 下载完成
- 下载失败
- 已取消

### 4.2 常见错误文案

- m3u8 无效或 token 已过期，请重新获取链接。
- 未找到 ffmpeg，请先安装：`brew install ffmpeg`
- 未找到 aria2c，请先安装：`brew install aria2`
- 分片下载未完成，请重试。
- ffmpeg 合并失败，请查看日志。
- ffprobe 验证失败，输出文件可能不完整。

## 5. 进度展示规则

- 解析到分片数量前，总进度显示不确定状态。
- 下载阶段使用分片完成比例：
  - `completedSegments / totalSegments`
- 合并阶段可显示固定文案，不强制显示百分比。
- 验证阶段通常很快，显示短暂状态即可。

## 6. 按钮行为

### 6.1 开始下载

点击后：

1. 校验输入。
2. 锁定输入字段。
3. 创建任务临时目录。
4. 进入 `Resolving`。

### 6.2 取消

点击后：

1. 终止正在运行的外部进程。
2. 状态切换为 `Cancelled`。
3. 保留已下载文件和日志。

### 6.3 重试

失败后点击开始下载视为重试：

1. 复用原输入。
2. 重新创建或清理当前任务临时目录。
3. 重新执行完整流程。

## 7. 完成态展示

完成后展示：

- 输出文件路径
- 文件大小
- 时长
- 视频编码和分辨率
- 音频编码
- 临时目录路径

示例：

```text
完成：/Users/uncle/Downloads/example.mp4
大小：180 MB
时长：18:01
视频：H.264 1280x720
音频：AAC
```
