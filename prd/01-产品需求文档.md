# Video Downloader 产品需求文档

版本：v1.0  
最后更新：2026-06-13  
适用范围：Tauri 桌面版 V1，本机自用，优先 macOS，架构预留 Windows/Linux

## 1. 背景

当前下载 HLS 视频需要手工执行多步命令：获取 m3u8、补全分片 URL、并发下载 ts 分片、按顺序合并为 mp4、再用 ffprobe 验证输出。这个流程可行但重复、容易出错，也不适合长期使用。

本产品目标是把上述流程封装为一个带 GUI 的桌面 App。当前技术路线调整为 Tauri：前端使用 Web UI，后端使用 Rust 调用本机下载与媒体工具。V1 先做稳定的 m3u8 下载器，未来再扩展为可以输入普通视频网页并自动发现 m3u8。

## 2. 目标用户

- 主要用户：本机桌面用户，当前优先 Mac 用户，熟悉复制 m3u8 链接，希望用图形界面完成下载。
- 使用环境：用户自己的电脑，当前不做公开分发，不进入 App Store。
- 典型输入：一个有效的 `.m3u8` URL。
- 典型输出：一个可播放的 `.mp4` 文件。

## 3. V1 产品目标

- 用户粘贴 m3u8 URL 后，可以一键下载、合并、验证并输出 mp4。
- 用户粘贴普通视频网页 URL 后，可以自动提取 m3u8 并进入下载流程。
- 下载过程可观察：显示当前阶段、分片数量、已完成数量、日志和错误。
- 常见失败可理解：token 无效、网络失败、工具缺失、分片下载失败、合并失败。
- 不要求用户手工写命令。

## 4. V1 非目标

- 不支持 DRM 破解、绕过登录授权或破解加密视频。
- 不支持批量任务队列。
- 不支持下载历史持久化。
- 不支持 App Store 发布、签名公证、自动更新。
- 不内置 ffmpeg、ffprobe、aria2c 二进制。

## 5. 核心用户流程

1. 用户打开 App。
2. 用户粘贴 m3u8 URL。
3. 用户选择输出目录，默认使用 `~/Downloads`。
4. 用户填写或接受默认文件名。
5. 用户点击“开始下载”。
6. App 检查本机工具依赖。
7. App 下载 m3u8，解析分片列表。
8. App 用 aria2c 并发下载分片。
9. App 用 ffmpeg 合并 mp4。
10. App 用 ffprobe 验证输出。
11. App 显示完成结果和输出路径。

## 6. 功能需求

### 6.1 URL 输入与校验

- 输入框接受 `http://` 或 `https://` URL。
- V1 只接受指向 m3u8 的 URL，允许 URL path 或 query 中包含 `.m3u8`。
- 空 URL、非 URL、非 m3u8 URL 要给出明确提示。

### 6.2 工具依赖检测

- 启动下载前检查：
  - `ffmpeg`
  - `ffprobe`
  - `aria2c`
- 优先查找系统 PATH。
- 若缺失，提示用户通过 Homebrew 安装：
  - `brew install ffmpeg aria2`

### 6.3 m3u8 获取与解析

- 下载远程 m3u8 到任务临时目录。
- 检测返回内容是否为有效 m3u8：
  - 必须包含 `#EXTM3U`
  - 必须包含至少一个 `#EXTINF`
- 若返回 JSON 错误、HTML 错误页或空内容，提示“m3u8 无效或 token 已过期”。
- 支持绝对分片 URL。
- 支持以 `/` 开头的相对分片 URL，并使用 m3u8 源 URL 的 scheme + host 补全。
- 支持普通相对路径，并以 m3u8 所在目录为 base 补全。

### 6.4 分片下载

- 生成 aria2c 输入清单。
- 分片按顺序命名：
  - `seg00000.ts`
  - `seg00001.ts`
  - `seg00002.ts`
- 使用多任务并发下载。
- V1 默认策略：
  - 多分片并发：`-j 32`
  - 单分片单连接：`-x 1 -s 1`
- 失败时允许重试。
- 下载完成后检查：
  - ts 文件数量等于 m3u8 分片数量
  - 没有残留 `.aria2` 状态文件

### 6.5 合并输出

- 生成 ffmpeg concat 清单。
- 使用无损合并，不重新编码。
- 输出格式为 mp4。
- 若输出文件已存在，V1 默认自动覆盖，界面需要显示会覆盖的目标路径。

### 6.6 输出验证

- 使用 ffprobe 获取：
  - duration
  - size
  - video codec
  - audio codec
  - width
  - height
- 验证成功后展示摘要。
- 验证失败时保留 mp4 并提示用户文件可能不完整。

### 6.7 日志

- 展示关键阶段日志：
  - 工具检测
  - 下载 m3u8
  - 分片数量
  - aria2c 下载进度
  - ffmpeg 合并结果
  - ffprobe 验证结果
- 日志应可复制。
- 失败时保留完整日志。

## 7. 异常处理需求

- token 无效：提示用户重新获取最新 m3u8。
- 网络超时：提示网络失败，允许重试。
- 分片缺失：提示缺失数量，允许重新下载。
- 工具缺失：提示缺失工具和安装命令。
- 合并失败：展示 ffmpeg 错误摘要，保留分片目录。
- 验证失败：提示输出文件可能不可播放。
- 用户取消：终止下载进程，保留临时目录供后续排查。

## 8. Roadmap

### V2：任务队列

- 支持多个 m3u8 任务排队。
- 支持下载历史。
- 支持任务重试和删除。

### V3：HTML 静态解析

状态：已进入 V1 实现。

- 用户输入普通视频网页 URL。
- App 下载 HTML。
- 从源码、script、video/source 标签中搜索 m3u8。

### V4：动态网页解析

- 状态：已进入 V1 实现。
- 使用 Playwright/Chromium 加载网页并捕获网络请求中的 m3u8。
- 使用 App 自己的持久浏览器 Profile 支持需要登录态的公开网页。
- 不做登录绕过、DRM 破解或未授权内容获取。

## 9. 验收标准

- 输入有效 m3u8 后可以生成可播放 mp4。
- 输入过期 token 返回明确错误。
- 缺少工具时不会开始任务，并给出安装建议。
- 分片数量和完成数量展示正确。
- 合并后能显示时长、大小、编码和分辨率。
- 失败时能保留日志，便于排查。
