# Video Downloader 测试验收计划

版本：v1.1  
最后更新：2026-06-13  
适用范围：Tauri 桌面版 V1，本机自用，优先 macOS

## 1. 单元测试

### 1.1 URL 校验

用例：

- 空字符串返回错误。
- 普通文本返回错误。
- `https://example.com/video.m3u8` 通过。
- `https://example.com/video.m3u8?token=abc` 通过。
- `https://example.com/watch/123` 在 V1 返回不支持。

### 1.2 m3u8 内容校验

用例：

- 包含 `#EXTM3U` 和 `#EXTINF` 通过。
- JSON 错误内容失败。
- HTML 错误页失败。
- 空内容失败。

### 1.3 分片解析

用例：

- 绝对分片 URL 保持不变。
- `/index.php/m3u8/ts/a.ts` 正确补全域名。
- `segments/a.ts` 正确基于 m3u8 目录补全。
- 分片顺序保持不变。

### 1.4 文件生成

用例：

- aria2 清单包含 URL 和 `out=seg00000.ts`。
- concat 清单包含有序 `file '/path/seg00000.ts'`。
- 分片序号从 0 开始。
- 5 位序号格式正确。

### 1.5 Tauri command

用例：

- `start_download` 能接收合法请求。
- `start_download` 对非法 URL 返回错误。
- `cancel_download` 在无任务时不会崩溃。
- `reveal_path` 对不存在路径返回可读错误。

## 2. 集成测试

### 2.1 有效 m3u8 下载

步骤：

1. 输入有效 m3u8 URL。
2. 选择 `~/Downloads`。
3. 开始下载。

期望：

- 能解析分片数量。
- 能下载全部分片。
- 能合并 mp4。
- ffprobe 返回媒体信息。

### 2.2 token 无效

步骤：

1. 输入已过期 token 的 m3u8 URL。
2. 开始下载。

期望：

- 不进入分片下载。
- 显示“m3u8 无效或 token 已过期”。
- 日志保留远端返回内容摘要。

### 2.3 工具缺失

步骤：

1. 临时让 App 找不到 `ffmpeg` 或 `aria2c`。
2. 开始下载。

期望：

- 不创建下载任务。
- 显示缺失工具名称。
- 显示 Homebrew 安装建议。

### 2.4 分片失败

步骤：

1. 使用会导致部分分片无法访问的 m3u8。
2. 开始下载。

期望：

- 下载结束后检测 `.aria2` 或缺失 ts。
- 状态为 Failed。
- 允许用户重试。

### 2.5 用户取消

步骤：

1. 开始下载较大的 m3u8。
2. 下载中点击取消。

期望：

- aria2c 进程被终止。
- 状态为 Cancelled。
- 日志保留。
- 临时目录不被自动删除。

## 3. 手工验收

### 3.1 基础流程

验收项：

- `npm run tauri dev` 可启动 GUI。
- URL 可以粘贴长链接。
- 输入框支持删除、复制、粘贴、剪切、替换和全选。
- 输出路径可以选择。
- 开始后输入控件锁定。
- 日志持续更新。
- 完成后可以在 Finder 中显示。
- mp4 可以播放。

### 3.2 真实样例

至少用一个真实 m3u8 完成端到端测试。

验收项：

- 分片数量与 m3u8 中 `#EXTINF` 数量一致。
- 输出文件大小合理。
- ffprobe 显示视频和音频流。
- 时长与预期接近。

### 3.3 体验检查

验收项：

- 错误提示能被非开发者理解。
- 日志能复制。
- 窗口尺寸下文本不重叠。
- 按钮启用状态符合任务状态。

## 4. 性能验收

目标：

- 200-300 个分片的视频能稳定完成。
- 合并阶段应在数秒到十几秒内完成，取决于文件大小。
- UI 在下载过程中不阻塞。

检查：

- 下载过程中窗口仍可拖动。
- 日志追加不会造成明显卡顿。
- 取消操作能在几秒内生效。

## 5. 回归清单

每次改动后检查：

- `npm run build` 通过。
- `cargo check` 通过。
- `npm run tauri dev` 可启动 GUI。
- 有效 m3u8 成功。
- token 无效失败。
- 工具缺失失败。
- 取消任务成功。
- 输出覆盖行为符合预期。
- 完成态媒体信息显示正常。
- 日志没有丢失关键错误。

## 6. V1 发布前验收标准

- 所有单元测试通过。
- 至少一个真实 m3u8 集成测试通过。
- 至少一个失败 m3u8 场景测试通过。
- App 在本机可通过 Tauri 启动。
- README 记录依赖和 V1 限制。
