# Video Downloader 技术方案

版本：v1.1  
最后更新：2026-06-13  
适用范围：Tauri 桌面版 V1，本机自用，优先 macOS，预留 Windows/Linux

## 1. 技术选型

- App 框架：Tauri 2
- 前端：HTML/CSS/JavaScript，当前不引入复杂前端框架
- 后端：Rust，通过 Tauri command 暴露能力
- 网页动态提取：Node.js + Playwright/Chromium
- 命令执行：Rust `std::process::Command`
- 下载工具：系统 `aria2c`
- 合并工具：系统 `ffmpeg`
- 验证工具：系统 `ffprobe`

V1 不内置二进制，只检测系统 PATH。当前本机已存在：

- `/usr/local/bin/ffmpeg`
- `/usr/local/bin/ffprobe`
- `/usr/local/bin/aria2c`

选择 Tauri 的原因：

- 同一套 Web UI 可复用到 Windows/Linux，避免后续重写多个原生客户端。
- Rust 后端适合处理进程编排、文件系统、URL 解析和任务状态。
- Tauri 安装包体积和资源占用低于 Electron，更适合本机工具。
- V1 继续复用已验证的命令行下载链路，降低核心下载风险。

## 2. 模块划分

### 2.1 Frontend AppShell

负责 Tauri Web UI：

- URL 输入
- 输出目录选择
- 文件名输入
- 状态展示
- 日志展示
- 开始、取消、打开文件、在文件管理器中显示
- 保证输入框支持键盘编辑、复制、粘贴、剪切、全选和替换

### 2.2 Tauri Bridge

负责前后端通信：

- `start_download`：自动判断输入 URL；m3u8 直接下载，普通网页先提取 m3u8 再下载
- `cancel_download`：取消当前任务
- `reveal_path`：在文件管理器中显示输出位置
- `download-log`：流式日志事件
- `download-progress`：分片进度事件
- `download-stage`：阶段变化事件

### 2.3 AppState / DownloadSession

负责描述和管理当前下载任务：

- 输入 URL
- 输出目录
- 输出文件名
- 临时目录
- 总分片数
- 已完成分片数
- 当前阶段
- 日志
- 错误信息
- ffprobe 验证结果
- 当前运行中的外部进程

V1 限制同一时间只运行一个下载任务。

### 2.4 M3U8Resolver

负责：

- 下载 m3u8 内容
- 校验 `#EXTM3U`
- 校验至少一个 `#EXTINF`
- 解析分片 URL
- 使用 `url::Url::join` 补全相对路径
- 生成 full m3u8
- 生成 aria2c 输入清单
- 生成 ffmpeg concat 清单
- 识别 HTML/JSON/空响应并提示 token 可能过期

### 2.5 SegmentDownloader

负责调用 aria2c：

```bash
aria2c --enable-rpc=false \
  -i aria2.txt \
  -d parts \
  -j 32 \
  -x 1 \
  -s 1 \
  -c \
  --max-tries=8 \
  --retry-wait=2 \
  --summary-interval=5
```

说明：

- `-j 32`：同时下载多个分片。
- `-x 1 -s 1`：每个分片单连接，避免部分 HLS 服务器 Range 响应不规范。
- `-c`：允许断点续传，但若出现 Range 错误，后续版本可清理失败分片后重试。

### 2.6 MediaMerger

负责调用 ffmpeg：

```bash
ffmpeg -hide_banner -y \
  -f concat \
  -safe 0 \
  -i concat.txt \
  -c copy \
  -bsf:a aac_adtstoasc \
  output.mp4
```

说明：

- 不重新编码。
- 速度快，质量无损。
- 输出 mp4。

### 2.7 MediaVerifier

负责调用 ffprobe：

```bash
ffprobe -hide_banner -v error \
  -show_entries format=duration,size \
  -show_entries stream=index,codec_type,codec_name,width,height \
  -of default=noprint_wrappers=1 \
  output.mp4
```

解析结果用于完成态展示。

### 2.8 ProcessRunner

统一封装外部命令：

- 启动进程
- 捕获 stdout
- 捕获 stderr
- 通过 Tauri event 流式追加日志
- 解析退出码
- 支持取消任务时 terminate

## 3. 数据流

1. 用户输入 m3u8 URL。
2. 前端调用 `start_download`。
3. Rust 后端创建任务目录：
   - `~/Downloads/VideoDownloaderTasks/{taskId}/`
4. 下载原始 m3u8：
   - `source.m3u8`
5. 解析分片并生成：
   - `full.m3u8`
   - `aria2.txt`
   - `concat.txt`
6. 下载分片到：
   - `parts/seg00000.ts`
   - `parts/seg00001.ts`
7. 合并输出：
   - 用户选择的 `output.mp4`
8. 验证输出并通过事件展示结果。

## 4. URL 补全规则

输入 m3u8 URL：

```text
https://example.com/path/video.m3u8?token=abc
```

分片处理：

- `https://cdn.example.com/a.ts` 保持不变。
- `/index.php/m3u8/ts/a.ts` 补为 `https://example.com/index.php/m3u8/ts/a.ts`。
- `segment/a.ts` 补为 `https://example.com/path/segment/a.ts`。

## 5. 文件命名规则

- 任务 ID：使用时间戳 + URL hash。
- 默认输出文件名：
  - 优先取 m3u8 basename
  - 若 basename 不可读，使用 `video-{yyyyMMdd-HHmmss}.mp4`
- 分片文件名固定 5 位序号：
  - `seg00000.ts`

## 6. 错误分类

- `InvalidURL`
- `MissingTool`
- `InvalidM3U8`
- `ExpiredToken`
- `SegmentDownloadFailed`
- `MergeFailed`
- `VerificationFailed`
- `Cancelled`

每类错误需要映射为用户可读文案。

## 7. HTML 网页提取 m3u8

### 7.1 静态提取

输入普通网页 URL 后：

1. 下载 HTML。
2. 从源码、script、video、source 标签中搜索 `.m3u8` 候选地址。
3. 对发现的候选 URL 做 base URL 补全。
4. 请求候选 m3u8 并验证 `#EXTM3U`。

### 7.2 动态提取

对前端 JS 动态生成视频地址的网站：

- 使用 Playwright/Chromium 加载页面。
- 监听 request/response URL。
- 读取 DOM 中的 video/source/script 标签和 performance resource entry。
- 使用 App 自己的持久浏览器 Profile 保存登录态。

### 7.3 抽象接口

```rust
trait VideoUrlExtractor {
    fn resolve(&self, input: &url::Url) -> Result<url::Url, String>;
}
```

V1 实现：

- `DirectM3U8Extractor`
- `StaticHtmlExtractor`
- `BrowserNetworkExtractor`

后续实现：

- `WebViewDomExtractor`

## 8. 安全边界

- V1 只处理普通 HLS。
- 不实现 DRM 绕过。
- 不破解密钥。
- 不自动使用浏览器 Cookie。
- 不规避网站访问控制。
